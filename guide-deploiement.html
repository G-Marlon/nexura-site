Configurer EmailJS
Requis Gratuit
ℹ️
EmailJS permet d'envoyer les messages du formulaire directement dans votre boîte email, sans aucun serveur backend. Gratuit jusqu'à 200 emails/mois.
Étape 1.1 — Créer un compte EmailJS
Rendez-vous sur emailjs.com et créez un compte gratuit avec votre email professionnel.
Étape 1.2 — Connecter votre service email
Dans EmailJS → Email Services → Add New Service → choisir Gmail (ou votre fournisseur) → Autoriser l'accès → Nommer le service service_smyllc
Étape 1.3 — Créer le template email
EmailJS → Email Templates → Create New Template
Nommer le template : template_contact

Copier ce contenu dans le template :
// Sujet du mail : Nouvelle demande SMYLLC — {{service_type}} de {{from_name}} // Corps du mail : Nom : {{from_name}} Email : {{from_email}} Service : {{service_type}} Message : {{message}} Répondre à : {{reply_to}}
Étape 1.4 — Récupérer la clé publique
EmailJS → Account → General → copier votre Public Key

Ouvrir SMYLLC-ultra-securise.html dans un éditeur texte et chercher :
// Chercher cette ligne (vers le début du script) : const EMAILJS_KEY = 'YOUR_PUBLIC_KEY'; // Remplacer par : const EMAILJS_KEY = 'VoTrEcLePuBLiQuE123'; // Votre vraie clé
⚠️
La clé publique EmailJS est visible dans le code HTML — c'est normal et voulu. Ne confondez pas avec votre clé privée qui, elle, ne doit jamais être exposée.
✅ Test EmailJS
Ouvrez le site → remplissez le formulaire de contact → envoyez → vérifiez votre boîte email dans les 2 minutes. Si le mail n'arrive pas, vérifiez le dossier Spam.
2
Déployer sur Netlify
Requis Gratuit
✅
Netlify est la méthode la plus simple. Déploiement en 30 secondes, HTTPS automatique, CDN mondial inclus. Plan gratuit largement suffisant pour SMYLLC.
Méthode A — Glisser-Déposer (Le plus rapide)
1. Aller sur app.netlify.com/drop
2. Créer un dossier smyllc-site/ contenant :
  • index.html (renommer SMYLLC-ultra-securise.html)
  • admin.html (renommer SMYLLC-admin.html)
3. Glisser ce dossier dans la zone de dépôt Netlify
4. Votre site est en ligne ! URL générée automatiquement.
⚠️
Renommer SMYLLC-ultra-securise.html en index.html est obligatoire — c'est le fichier que Netlify charge par défaut.
Méthode B — Via GitHub (Recommandé pour les mises à jour)
1. Créer un compte GitHub (gratuit)
2. Nouveau repository → nommer smyllc-website → Public ou Private
3. Uploader les fichiers dans le repo
4. Sur Netlify → New site from Git → choisir le repo
5. Chaque fois que vous modifiez un fichier sur GitHub → le site se met à jour automatiquement
Créer le fichier _redirects (obligatoire pour la navigation SPA)
Créer un fichier nommé exactement _redirects (sans extension) dans le même dossier que index.html :
/* /index.html 200
Ce fichier dit à Netlify de toujours servir index.html pour toutes les URLs — indispensable pour le site SPA (Single Page Application).
Créer le fichier netlify.toml (sécurité HTTPS)
Créer un fichier netlify.toml dans le dossier du site :
[[headers]] for = "/*" [headers.values] X-Frame-Options = "DENY" X-Content-Type-Options = "nosniff" Referrer-Policy = "strict-origin-when-cross-origin" Permissions-Policy = "geolocation=(), microphone=(), camera=()" Strict-Transport-Security = "max-age=31536000; includeSubDomains; preload" [[redirects]] from = "/*" to = "/index.html" status = 200
Ce fichier configure les en-têtes de sécurité HTTPS directement au niveau du serveur, renforçant encore la protection du site.
3
Google Analytics 4
Optionnel Gratuit
ℹ️
GA4 est déjà intégré dans le code mais désactivé. Il s'active automatiquement après le consentement cookie du visiteur. Respect RGPD garanti.
Configurer GA4 en 3 étapes
1. Aller sur analytics.google.com
2. Créer une propriété → saisir smyllc.ci → obtenir l'ID de mesure (format : G-XXXXXXXXXX)
3. Dans index.html, chercher et remplacer :
// Chercher : data-ga-id="G-XXXXXXXXXX" // Remplacer par votre vrai ID : data-ga-id="G-ABC123DEF4"

Connecter le domaine à Netlify
1. Netlify → votre site → Domain settings → Add custom domain
2. Saisir smyllc.ci → Netlify affiche 2 DNS records à configurer
3. Chez votre registrar → DNS Management → ajouter les enregistrements :

Type A : smyllc.ci → IP fournie par Netlify
Type CNAME : www.smyllc.ci → votre-site.netlify.app

4. Propagation DNS : 15 min à 48h
5. Netlify active HTTPS automatiquement (Let's Encrypt)
Mettre à jour la canonical URL dans index.html
Chercher et remplacer dans index.html :
// Avant : <link rel="canonical" href="https://smyllc.ci/"/> // Vérifier que smyllc.ci est bien votre domaine final // Remplacer aussi dans le JSON-LD : "url": "https://smyllc.ci"
5
Brevo — Newsletter
Recommandé Gratuit 300/j
✅
Brevo (ex-Sendinblue) est gratuit jusqu'à 300 emails/jour et contacts illimités. Parfait pour démarrer.
Configurer Brevo en 4 étapes
1. Créer un compte sur brevo.com
2. Contacts → Lists → créer une liste "Newsletter SMYLLC"
3. SMTP & API → API Keys → créer une clé API
4. Dans index.html, trouver la fonction nlSubmit et remplacer le setTimeout(doSuccess, 800) par :
// Remplacer le setTimeout de simulation par : fetch('https://api.brevo.com/v3/contacts', { method: 'POST', headers: { 'accept': 'application/json', 'api-key': 'VOTRE_CLE_API_BREVO', 'content-type': 'application/json' }, body: JSON.stringify({ email: email, listIds: [1], // ID de votre liste Brevo updateEnabled: true }) }) .then(r => r.ok ? doSuccess() : doError()) .catch(doError);

Remplacer les Photos Unsplash
Priorité 1
🚨
C'est l'amélioration la plus impactante ! Toutes les photos actuelles sont des placeholders Unsplash. Les remplacer par vos vraies photos SMYLLC est indispensable pour la crédibilité.
Via le panneau Admin (recommandé)
1. Ouvrir SMYLLC-admin.html
2. Se connecter (admin / SMYLLC@2025! → changer après !)
3. Aller dans Photos & Portfolio
4. Glisser-déposer vos photos ou cliquer "Ajouter des Photos"
5. Assigner la catégorie (Portrait, Mariage, Mode...)
6. Publier → les photos apparaissent dans la galerie

7
Sécuriser le Panneau Admin
Urgent
🚨
Changer les identifiants par défaut immédiatement après la première connexion. Ne jamais déployer le panneau admin avec les credentials par défaut.
Changer les identifiants admin
1. Ouvrir SMYLLC-admin.html
2. Se connecter avec : admin / SMYLLC@2025!
3. Aller dans Paramètres → Sécurité & Accès
4. Saisir un nouveau identifiant et un nouveau mot de passe fort
5. Cliquer "Mettre à jour les identifiants"
⚠️
Règles pour un bon mot de passe admin : 16+ caractères, majuscules + minuscules + chiffres + symboles. Ex: Sm!LLC_2025#Ph0t0
Accès admin en production
L'admin est accessible à l'URL : https://smyllc.ci/admin.html

Pour une sécurité maximale en production, envisager :
• Renommer admin.html en un nom aléatoire : gestion-7x9k2.html
• Ajouter une protection par mot de passe au niveau Netlify (Netlify Identity)
• Restreindre l'accès par IP si vous avez une IP fixe

SEO — Actions Finales
Important
Créer le sitemap.xml
Créer le fichier sitemap.xml dans le dossier du site :
<?xml version="1.0" encoding="UTF-8"?> <urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9"> <url><loc>https://smyllc.ci/</loc><priority>1.0</priority></url> <url><loc>https://smyllc.ci/#about</loc><priority>0.8</priority></url> <url><loc>https://smyllc.ci/#services</loc><priority>0.9</priority></url> <url><loc>https://smyllc.ci/#portfolio</loc><priority>0.8</priority></url> <url><loc>https://smyllc.ci/#contact</loc><priority>0.7</priority></url> </urlset>
Créer le robots.txt
Créer le fichier robots.txt :
User-agent: * Allow: / Disallow: /admin.html Sitemap: https://smyllc.ci/sitemap.xml
Soumettre à Google Search Console
1. search.google.com/search-console → Ajouter propriété → saisir https://smyllc.ci
2. Vérifier la propriété (méthode DNS recommandée)
3. Sitemaps → soumettre https://smyllc.ci/sitemap.xml
4. Attendre 2-7 jours pour l'indexation Google
Créer une fiche Google Business Profile
1. business.google.com → Ajouter votre entreprise
2. Saisir : SMYLLC Photography Studio · Cocody Riviera 2, Abidjan
3. Catégorie principale : Photographe
4. Ajouter photos, horaires, description
5. Recueillir des avis → améliore énormément le référencement local
